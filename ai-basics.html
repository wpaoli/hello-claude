Term Definition Examples Why It Matters
Context The information the AI has available when making a decision or taking an action Chat history, system instructions, retrieved docs (RAG example), memory (OpenAI overview) Context drives accuracy. Too little → wrong answers. Too much → noise + token limits. In agents, bad context = bad decisions.
MCP (Model Context Protocol) A structured way to pass context and tools to the AI in a consistent, controlled way Model Context Protocol, structured tool + context passing Keeps systems predictable. Without this, agents become messy and hard to debug.
Prompts The instructions or guidance given to the AI Prompt engineering guide, "Act as an incident commander…" Prompts shape behavior, but can't fix bad context. Don't over-index here.
Tools External functions the AI can call to take real actions or fetch fresh data Function calling (OpenAI tools), API calls, DB queries This is what makes agents actually useful. Otherwise it's just text generation.
Scaffolding The surrounding system that controls how the AI operates LangChain agents, workflows, retries, guardrails This is the difference between a demo and a production system.
Tokens Units of text the model processes; they limit how much context you can send/receive Tokenizer, long vs short prompts Hard constraint. Impacts cost, speed, and accuracy. Overflow = silent context loss.
Skills The capabilities the AI is enabled to perform Code generation, summarization, tool usage, reasoning tasks Defines what's possible — but results depend on context + structure.
